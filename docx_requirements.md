产品核心功能框架

整体架构

小程序分为**学生端**、**老师端**，后端配套**管理后台**；通过用户登录时的身份选择与校验，自动匹配对应操作页面与功能权限，核心围绕**比赛全生命周期管理**+**积分体系**+**消息沟通**+**作品线上评审**+**数据可视化**五大核心板块搭建，实现功能全覆盖、流程全打通。

各端核心功能

1.  **学生端**：登录注册、个人信息管理、比赛浏览 / 报名 / 签到、比赛作品上传、比赛结果 / 奖状查看、积分查询 / 任务获取 / 奖品兑换、站内私信（一对一 / 群聊）、每日签到、比赛分享

2.  **老师端**：包含学生端全部功能 + 比赛发布 / 编辑 / 下架、参赛人员管理、比赛签到审核、比赛作品线上评审、评分评等级、积分发放、奖状 / 排名发布、个人比赛数据大屏、比赛群消息发送

3.  **管理后台**：师生信息管理、多角色权限管理、全局比赛管理、积分体系全配置、奖品管理、订单管理、系统日志管理、汇总数据大屏、轮播图配置、系统功能开关、数据统计与报表导出、多校区管理

需求详情

一、基础功能模块 - 登录与注册

核心需求

实现基于学号的专属注册登录，完成身份（学生 / 老师）判定，支持密码找回与账号注销，为后续权限分配做基础。

功能详情

1.  **注册**

    -   输入项：唯一学号（校园专属标识，做唯一性校验）、姓名、手机号（短信验证码校验）、身份选择（学生 / 老师）、设置登录密码

    -   校验规则：学号不能为空且未被注册；短信验证码有效期 5 分钟，仅支持对应手机号验证；密码长度 8-16 位，含数字 + 字母组合

    -   注册成功：自动跳转登录页面，提示注册成功并绑定身份标签，同步生成用户基础信息档案

2.  **登录**

    -   登录方式：学号 + 密码登录（支持记住密码，小程序端有效期 7 天，可手动清除）

    -   校验规则：学号与密码匹配、身份标签与系统记录一致

    -   登录后逻辑：根据身份自动跳转对应端首页（学生端 / 老师端），保留登录状态，展示未读消息提醒

3.  **辅助功能**

    -   忘记密码：通过手机号 + 短信验证码重置密码，仅支持注册时绑定的手机号，重置后需重新登录

    -   账号注销：支持用户手动注销，注销前需二次确认，注销后所有个人数据（除比赛历史记录）永久清除，账号不可恢复

二、基础功能模块 - 个人信息管理

核心需求

支持师生完善、修改个人信息，可视化展示个人核心数据（积分、比赛、奖品、作品等），实现个性化设置与隐私管理。

功能详情

1.  **信息展示与编辑**

    -   基础信息：头像（支持微信头像一键上传 / 本地自定义图片上传，支持裁剪）、姓名、学号、身份、手机号（可修改，需短信验证）、所属校区 / 年级 / 专业（学生端）、所属科室 / 教研组（老师端）

    -   编辑规则：学号为唯一标识，不可修改；所有信息修改后需保存生效，实时同步至系统与个人档案，管理员可查看修改记录

2.  **个人数据中心（可视化展示）**

    -   学生端：我的积分（当前可用积分 + 积分获取 / 消耗趋势图）、我的比赛（已报名 / 进行中 / 已结束 / 获奖记录，按时间分类）、我的作品（已上传 / 待评审 / 已评审，支持查看评审意见）、我的奖品（已兑换 / 待领取 / 已领取 / 已作废，展示奖品详情与领取进度）、我的订单（积分兑换订单，展示订单号 / 状态 / 兑换时间）

    -   老师端：我的积分（系统发放 / 奖励积分记录）、我发起的比赛（草稿 / 待开始 / 进行中 / 已结束，展示各比赛核心数据）、我评分的比赛（待评分 / 已评分 / 待发布结果）、我的评审作品（待评审 / 已评审，按比赛分类）、个人数据大屏（发起比赛数、参与率、获奖率等核心数据可视化）

3.  **个性化设置**

    -   消息提醒：可开启 / 关闭比赛报名成功、比赛开始、作品评审、评分结果、积分发放、奖品兑换、私信等消息通知，支持单独设置微信服务通知权限

    -   隐私设置：可设置个人比赛记录、积分信息、作品信息是否公开，可设置是否允许其他用户发起私信

    -   通用设置：清除小程序缓存、修改登录密码、关于小程序、意见反馈（支持文字 + 图片提交）、客服联系方式（一键跳转）

三、核心业务模块 - 比赛管理（老师端核心，全功能版）

核心需求

为老师提供比赛发布、编辑、下架、人员管理、签到管理、作品评审的全流程工具，支持自定义比赛所有规则与设置，实时把控比赛全进度。

功能详情

1.  **比赛发布（全字段自定义）**

    -   发布表单支持草稿保存（未发布比赛可在 “草稿箱” 编辑 / 发布 / 删除，草稿永久保存），需填写以下信息：

        -   基础信息：比赛名称、比赛封面图（支持上传）、比赛说明（富文本编辑器，支持图片 / 文字 / 排版 / 超链接）、比赛难度（简单 / 中等 / 困难，支持自定义难度名称）、比赛重要性（校级 / 院级 / 班级 / 社团 / 教研组，支持自定义）

        -   时间地点：比赛开始时间、结束时间、签到开始 / 截止时间、作品上传开始 / 截止时间、评审开始 / 截止时间、比赛具体地址（支持地理位置选择 / 手动输入，生成定位，支持导航）

        -   参与设置：比赛名额（可设置不限名额）、报名开始 / 截止时间、参与对象（全体学生 / 指定校区 / 指定年级 / 指定专业 / 指定班级，支持多条件筛选）、报名需提交信息（可自定义，如姓名 / 学号 / 联系方式 / 作品简介等）

        -   作品设置：是否需要上传作品、作品类型（文档 / 图片 / 视频 / 音频，支持多类型选择）、作品大小限制、作品命名规则、是否支持多次上传（仅保留最新版）

        -   奖励设置：各排名等级（一等奖 / 二等奖 / 三等奖 / 优秀奖 / 参与奖等，支持自定义等级名称 / 数量 / 分数线）、对应奖励积分、奖状模板选择（系统提供多套基础模板，支持自定义奖状文字 / LOGO / 背景）、实物奖励设置（可关联积分商城奖品）

    -   发布规则：填写必填项（带 \*）后方可发布；发布后比赛状态为 “待开始”，同步至小程序比赛列表与首页推荐（符合推荐规则），学生可查看并报名

2.  **比赛编辑与下架**

    -   编辑权限：比赛报名截止前，老师可修改所有比赛信息；报名截止后，仅可修改比赛时间、地点、比赛说明、评审时间，不可修改参与名额、参与对象、作品设置、奖励设置

    -   编辑后同步：修改内容实时同步至学生端比赛详情页，若修改关键信息（时间 / 地点 / 作品上传规则），自动向已报名学生发送**群私信 + 微信服务通知**双重提醒

    -   下架功能：比赛发布后，未开始前可手动下架，下架后比赛从列表中隐藏，已报名学生收到通知，可选择取消报名或等待重新上架；比赛开始后不可下架，仅管理员可强制下架

3.  **参赛人员管理**

    -   查看列表：展示已报名学生的所有信息（姓名 / 学号 / 所属信息 / 报名时间 / 提交的报名信息 / 是否上传作品 / 是否签到），支持按学号 / 姓名 / 报名时间 / 专业筛选、搜索，支持导出（Excel 格式，可选择导出字段）

    -   操作功能：比赛开始前，可驳回不符合参与条件的学生报名（需填写驳回原因，自动私信 + 微信通知学生）；可手动添加参赛学生（针对特殊情况）；比赛开始后，可标记学生 “缺席 / 到场”

4.  **比赛签到管理（多方式支持）**

    -   签到方式：支持**线下扫码签到**（老师生成专属签到二维码，小程序端展示，可刷新）、**地理位置签到**（学生需在比赛定位范围内完成签到，支持设置定位半径）、**手动签到**（老师端直接标记学生签到）

    -   老师端操作：可查看实时签到记录（签到学生 / 未签到学生，展示签到时间 / 签到方式），支持批量标记签到、关闭签到功能（签到截止后自动关闭）；签到截止后自动生成签到统计报表（含签到率 / 各专业签到情况），支持导出

5.  **比赛作品管理与评审**

    -   作品查看：按比赛分类展示所有学生上传的作品，支持在线预览（文档 / 图片 / 视频 / 音频）、下载作品，展示作品上传时间 / 文件名 / 文件大小

    -   作品评审：支持**批量评审**（按等级 / 分数批量标记作品）、**单个评审**（为作品单独设置分数 / 等级，支持填写评审意见，支持上传评审批注图片）；支持设置**多评委评审**（老师可邀请其他老师作为评委，分配评审权限），系统自动计算平均分

    -   评审结果：评审完成后，可在老师端查看所有作品的评审分数 / 等级 / 评审意见，支持导出评审结果表，可选择是否向学生展示评审意见

四、核心业务模块 - 比赛参与（学生端核心，全功能版）

核心需求

为学生提供比赛浏览、报名、签到、作品上传、结果查看的一站式入口，支持比赛分享、每日签到赚积分，清晰展示比赛所有关键信息与个人参赛进度。

功能详情

1.  **比赛浏览（精准筛选 + 个性化推荐）**

    -   比赛列表：首页 / 比赛专区展示所有待开始 / 进行中比赛，支持**多维度排序**（最新发布 / 比赛时间 / 重要性 / 报名人数），支持**精准筛选**（难度 / 重要性 / 参与对象 / 是否需要作品 / 所属校区），支持关键词搜索比赛名称 / 发起人；列表展示比赛封面 / 名称 / 难度 / 重要性 / 剩余名额 / 报名倒计时 / 发起人

    -   比赛推荐：首页根据学生所属年级 / 专业 / 历史报名记录，个性化推荐匹配的比赛，标注 “为你推荐”

    -   比赛详情：展示比赛所有信息（基础信息 / 时间地点 / 参与设置 / 作品设置 / 奖励设置 / 发起人 / 已报名人数 / 剩余名额），发起人显示老师姓名 / 头像 / 所属信息，支持点击查看老师主页（仅展示公开信息）；支持收藏比赛、分享比赛（微信好友 / 朋友圈，分享可赚积分）

2.  **比赛报名与取消**

    -   报名条件：在报名时间内，符合参与对象要求，比赛名额未满

    -   报名操作：点击 “立即报名”，按老师要求填写报名信息（自定义项），二次确认后报名成功，扣减对应比赛名额，自动生成报名凭证（小程序内展示，支持截图），同步发送报名成功通知

    -   报名取消：报名截止前，学生可手动取消报名，名额自动恢复；报名截止后不可取消，若无故未参与 / 未签到，按管理后台规则扣除少量积分，同步发送扣除通知并说明原因

3.  **比赛签到**

    -   签到入口：比赛详情页 / 个人中心 “我的比赛 - 进行中” 醒目展示签到入口，签到时间内显示 “立即签到”，超时显示 “签到已截止”，展示签到剩余时间

    -   签到操作：按老师设置的方式完成签到，签到成功后提示 “签到成功” 并展示签到凭证，同步至老师端签到记录；若为地理位置签到，超出定位范围将提示 “未在比赛指定区域，无法签到”

4.  **比赛作品上传**

    -   上传入口：比赛详情页 / 个人中心 “我的比赛 - 进行中” 展示作品上传入口，上传时间内显示 “立即上传”，超时显示 “上传已截止”

    -   上传操作：按老师要求选择作品类型，上传符合大小 / 命名规则的作品，支持预览、重新上传（仅保留最新版）；上传成功后提示 “作品上传成功”，同步至老师端作品列表，支持查看上传记录

    -   补充说明：可在上传作品时填写作品简介，支持后期编辑（上传截止前）

5.  **比赛结果与奖状查看（全流程展示）**

    -   结果展示：比赛结束后，老师发布评分排名后，学生可在比赛详情页 / 个人中心查看**比赛排名名单**（按等级展示，含姓名 / 学号 / 分数 / 奖项）、个人评分 / 等级 / 评审意见（若老师开启展示）、对应奖励积分（积分到账后同步提示）

    -   奖状查看：获奖学生可在个人中心 / 比赛详情页查看电子奖状（带比赛信息 / 个人信息 / 奖项），支持**保存至手机**/**分享至微信好友 / 朋友圈**/**自定义奖状样式**（少量个性化设置）

    -   获奖记录：所有获奖记录永久保存在个人中心，支持随时查看、截图、分享

五、核心业务模块 - 比赛评分与奖励发放（老师端）

核心需求

实现老师对参赛学生的线上评分评等级（支持多维度 / 多评委），自动 / 手动发放积分，发布公开的排名、奖状与名单，支持实物奖励关联。

功能详情

1.  **评分评等级（多维度自定义）**

    -   评分入口：老师端 “我发起的比赛 - 已结束” 选择对应比赛，进入评分页面（若有作品，需完成作品评审后进入）

    -   评分设置：支持自定义**评分维度**（如作品质量 / 现场表现 / 创意性 / 完成度等，可设置多个维度，各维度占比可自定义），系统自动根据维度占比计算总分；支持设置**等级与分数段绑定**（如 90 分及以上为一等奖，可自定义分数段与等级）

    -   评分方式：支持**批量评分**（按等级批量标记学生，自动匹配对应分数）、**单个评分**（为学生单独设置各维度分数 / 总分数 / 等级，支持自定义分数）；支持**多评委评审**，系统自动计算所有评委的平均分，按平均分判定等级；评分表支持导出 / 导入（Excel 格式）

2.  **积分发放（自动 + 手动，全场景覆盖）**

    -   自动发放：老师发布评分排名后，系统根据预设的 “等级 - 积分” 对应关系，**自动为获奖学生发放奖励积分**，同步至学生积分账户，发送积分到账通知（私信 + 微信），积分记录标注 “XX 比赛 XX 奖项奖励”

    -   手动发放：支持老师为**所有参与学生发放参与积分**（无等级学生，可设置固定积分 / 自定义积分），支持批量 / 单个发放；支持为表现优秀的学生发放**额外奖励积分**，需填写发放原因；发放记录实时展示，支持导出

    -   积分撤回：积分发放后，若出现错误，老师可在 24 小时内撤回积分（需填写撤回原因），同步通知学生，撤回记录计入系统日志

3.  **排名与奖状发布（多渠道通知）**

    -   排名发布：老师完成评分后，可预览排名名单，确认无误后点击 “发布排名”，排名名单与学生个人评分 / 等级 / 评审意见同步至学生端，所有用户可查看公开排名；支持发布前编辑排名（仅可修改分数 / 等级，需填写修改原因）

    -   奖状发布：系统根据比赛信息与学生获奖等级，**自动生成电子奖状**，老师可在发布前预览奖状样式（支持批量预览 / 单个预览），确认无误后发布；发布后学生可查看 / 保存 / 分享奖状，支持老师重新生成奖状（针对样式错误）

    -   多渠道通知：排名与奖状发布后，系统自动向**所有参赛学生发送群私信 + 微信服务通知**，告知比赛结果、积分发放情况、奖状查看入口、评审意见查看入口

六、核心业务模块 - 积分体系（全功能版，任务 + 兑换 + 管理）

核心需求

搭建完善的**积分获取 - 积分消耗 - 积分管理**体系，支持多场景积分获取、丰富的积分兑换奖品、精细化的积分管理，提升学生参赛积极性。

功能详情

1.  **积分获取（多场景，自动 + 手动）**

    -   比赛奖励类（核心）：比赛获奖积分（老师设置并自动 / 手动发放）、比赛参与积分（老师手动发放）、优秀作品额外积分（老师手动发放）

    -   日常任务类（自动）：每日签到（小程序端，每日一次，固定积分，连续签到可叠加积分，管理后台可配置）、比赛分享（每日限 1 次，分享至微信好友 / 朋友圈，固定积分，自动发放）、首次完善个人信息（一次性积分，自动发放）

    -   积分记录：学生端 / 老师端 / 管理后台均可查看所有积分获取 / 消耗记录，包含**获取 / 消耗时间 / 原因 / 积分数量 / 操作人 / 关联比赛**，支持按时间 / 类型筛选，记录永久保存，支持导出

2.  **积分消耗（奖品兑换 + 违规扣除）**

    -   奖品兑换（核心）：学生在积分商城用积分兑换奖品，兑换成功后**自动扣除对应积分**，积分不足无法兑换；支持积分 + 现金兑换（管理后台可配置）

    -   违规扣除（自动 + 手动）：无故未参与已报名比赛、无故未签到、比赛作弊、作品抄袭等违规行为，**按管理后台规则自动扣除积分**；管理员可手动对违规学生扣除积分（需填写扣除原因，计入系统日志）；扣除积分后同步发送通知，说明扣除原因与扣除数量

3.  **积分商城（学生端，全功能）**

    -   奖品展示：按**积分高低 / 奖品类型 / 热门程度**展示奖品，奖品分类清晰（学习用品 / 生活用品 / 文创产品 / 实物奖品 / 虚拟奖品 / 优惠券），每个奖品标注**奖品名称 / 所需积分 / 库存数量 / 兑换说明 / 奖品图片 / 领取方式 / 兑换限制**（如每人限兑 1 次）

    -   兑换操作：点击 “立即兑换”，二次确认后兑换成功，**自动生成兑换订单**（含订单号 / 兑换时间 / 奖品信息 / 所需积分），扣减对应积分与奖品库存；若奖品为积分 + 现金，需完成支付后生成订单

    -   奖品领取：支持**线下领取**（标注领取地址 / 领取时间 / 领取联系人 / 联系方式，支持导航）、**线上邮寄**（学生需填写收货地址 / 电话 / 姓名，支持修改收货地址，管理员可录入物流信息，学生可查看物流进度）、**虚拟奖品自动发放**（如优惠券 / 会员，兑换成功后立即发放至个人中心）

    -   订单管理：学生端可查看所有兑换订单（待支付 / 待领取 / 已领取 / 已作废 / 物流中），支持取消待支付订单（积分自动退还）、查看订单详情、申请售后（针对质量问题，需上传图片）

4.  **积分管理（管理后台核心，精细化配置）**

    -   积分规则配置：可配置**所有日常任务的积分数量**（每日签到 / 比赛分享 / 首次完善信息）、**违规行为的扣除积分数量**、连续签到叠加积分规则、积分有效期（可设置永久有效 / 固定有效期）

    -   积分调整：管理员可对任意学生 / 老师的积分进行**手动增加 / 减少**（需填写调整原因，计入系统日志），支持批量调整，调整后同步通知用户

    -   积分统计：可视化展示平台**总发放积分 / 总消耗积分 / 剩余可用积分**、学生人均积分、积分获取 / 消耗趋势图（按日 / 周 / 月 / 年）、各渠道积分获取 / 消耗占比，支持按校区 / 年级 / 专业统计积分分布，所有统计数据支持导出（Excel / 图片）

    -   积分冻结：针对违规学生，管理员可冻结其积分（冻结后无法兑换奖品，需填写冻结原因），解冻后积分恢复可用，冻结 / 解冻记录计入系统日志

七、核心业务模块 - 站内私信（全功能版，一对一 + 群聊 + 系统消息）

核心需求

实现小程序内**一对一私信 + 比赛群聊 + 系统消息**全场景沟通，满足老师与学生的比赛通知、答疑、互动需求，无需跳转其他社交平台，支持多渠道提醒。

功能详情

1.  **消息类型（全覆盖）**

    -   系统消息：注册 / 登录成功、报名成功 / 取消、签到成功、作品上传成功 / 评审意见、积分发放 / 消耗 / 扣除、奖品兑换 / 订单状态变更、比赛信息修改 / 下架等，系统自动发送，有专属系统头像，消息内容带相关比赛 / 订单 / 积分链接，可一键跳转

    -   个人私信：老师与学生、学生与学生之间的一对一聊天，支持**文字 / 表情 / 图片 / 文件**发送（文件大小有限制），支持查看聊天对象的公开个人信息，支持一键发起比赛咨询

    -   比赛群私信：老师为每场比赛创建专属群聊，所有已报名学生自动加入，老师可发送**文字 / 表情 / 图片 / 文件 / 比赛通知**，支持撤回消息（发送后 5 分钟内）、清空群聊记录、移除群内成员（针对违规学生）；学生可在群内提问、互动，支持 @老师 / 其他学生

2.  **私信功能（全体验）**

    -   聊天列表：按**最新消息时间**排序，展示聊天对象（老师 / 学生 / 系统 / 比赛群）、最新消息内容、未读消息数量（红色角标）、最后消息时间、群聊人数（比赛群），支持搜索聊天对象，支持置顶聊天（一对一 / 群聊）

    -   未读提醒：新消息收到后，**小程序端红点提醒 + 微信服务通知提醒**（用户可在设置中关闭），进入聊天页面后自动标记为已读，支持手动标记已读 / 未读

    -   聊天记录：保存所有聊天记录（文字 / 图片 / 文件 / 表情），支持下拉加载更多，支持删除单条 / 全部聊天记录，支持收藏重要消息；文件支持在线预览 / 下载，图片支持保存至手机

    -   便捷咨询：学生可在任意比赛详情页点击 “咨询发起人”，**直接跳转到与该比赛老师的一对一私信聊天页面**，快速发起咨询，聊天框自动附带比赛名称

3.  **消息管理（全权限）**

    -   个人管理：支持删除聊天、屏蔽聊天对象（屏蔽后不再接收该对象的私信 / 群消息，支持取消屏蔽）、设置聊天免打扰（仅红点提醒，无微信服务通知）、清空所有未读消息

    -   群聊管理（老师端）：比赛群聊创建者（老师）可修改群聊名称、发布群公告（置顶展示）、移除群内成员、清空群聊记录、解散群聊（比赛结束后可手动解散，或系统自动解散）

    -   系统消息管理：支持一键清空系统消息，保留近 1 年的系统消息记录，支持按类型筛选系统消息（比赛 / 积分 / 奖品 / 订单）

八、核心展示模块 - 数据大屏（全端覆盖，可视化）

核心需求

为**管理员 / 老师**分别打造专属数据大屏，以可视化图表展示平台 / 个人比赛核心数据，支持数据筛选、刷新、导出，实现数据化管理与决策。

功能详情

1.  **大屏展示位置与权限**

    -   管理后台大屏：首页默认展示**平台全局汇总数据大屏**，所有管理员均可查看，支持全屏查看，权限最高

    -   老师端大屏：个人中心提供**个人比赛数据大屏**入口，仅老师可查看自己发起的比赛相关数据，支持全屏查看

2.  **管理后台全局汇总大屏（全维度数据）**

    -   基础数据概览：平台注册学生数 / 老师数 / 总用户数、各校区用户分布、新增用户趋势（按日 / 周 / 月）

    -   比赛数据：平台总比赛数、待开始 / 进行中 / 已结束比赛数、各难度 / 重要性比赛占比（饼图）、各校区 / 教研组发起比赛数（柱状图）、老师发起比赛数 TOP 排名（柱状图）、场均报名人数 / 参与率 / 获奖率

    -   参与数据：总报名人数、各比赛报名人数 TOP 排名（折线图）、各年级 / 专业学生参与率（热力图）、比赛签到率整体统计、作品上传率整体统计

    -   积分数据：总发放积分 / 总消耗积分 / 剩余可用积分、积分获取 / 消耗趋势图（折线图）、各渠道积分获取 / 消耗占比（饼图）、学生人均积分、积分兑换率

    -   奖品与订单数据：积分商城奖品总数 / 各类型奖品占比、总兑换订单数 / 订单完成率、各奖品兑换数 TOP 排名（柱状图）、线下 / 线上领取订单占比、订单趋势图

3.  **老师端个人比赛数据大屏（个性化数据）**

    -   个人比赛概览：发起比赛总数、草稿 / 待开始 / 进行中 / 已结束比赛数、各难度 / 重要性比赛占比

    -   比赛参与数据：场均报名人数、总报名人数、各比赛报名人数排名、比赛签到率 / 作品上传率 / 评审完成率

    -   比赛结果数据：各比赛获奖率、各奖项发放数量、总发放积分数量、各比赛积分发放数量排名

    -   互动数据：学生咨询消息数、未读消息数、比赛群聊消息数、各比赛学生互动率

4.  **大屏通用功能（全支持）**

    -   数据筛选：支持按**时间范围**（今日 / 本周 / 本月 / 本季度 / 本年 / 自定义）筛选所有数据，支持按**比赛类型 / 校区**筛选（管理后台）

    -   数据刷新：支持**手动刷新**（点击刷新按钮）、**自动刷新**（默认 5 分钟一次，可在设置中修改刷新频率）

    -   数据导出：支持将大屏所有数据 / 单个图表数据导出为**Excel / 图片 / PDF**格式，支持选择导出字段

    -   数据钻取：点击图表可查看下一级明细数据（如点击报名人数 TOP 排名，可查看该比赛的详细报名信息）

九、后端管理后台模块（全功能版，总控中心）

核心需求

为平台管理员提供**师生管理、多角色权限管理、全局比赛管理、积分奖品管理、系统管理、数据统计**的一站式总控后台，支持精细化、智能化管理，保障小程序稳定运行。

功能详情

1.  **师生信息管理（精细化）**

    -   信息查看：展示所有注册学生 / 老师的**完整个人档案**（基础信息 / 所属信息 / 积分信息 / 比赛记录 / 奖品记录 / 订单记录 / 违规记录），支持按学号 / 姓名 / 手机号 / 校区 / 年级 / 专业 / 注册时间筛选、搜索，支持模糊搜索

    -   操作功能：支持**冻结 / 解冻账号**（冻结后账号无法登录，需填写原因，计入日志）、**修改用户身份**（学生→老师 / 老师→学生，需填写原因）、**重置用户密码**（重置后发送临时密码至用户手机号）、**完善用户信息**（针对信息缺失用户）、**标记违规用户**（计入违规记录，可关联扣分 / 冻结积分）

    -   数据管理：支持导出师生信息列表（Excel 格式，可选择导出字段）、导入师生信息（批量添加，支持模板导入）、删除无效用户（未注册 / 未登录超 1 年，需二次确认）

2.  **多角色权限管理（全维度）**

    -   角色管理：默认角色为**学生 / 老师 / 超级管理员**，支持**新增自定义角色**（如院级管理员 / 班级管理员 / 奖品管理员 / 数据管理员），支持修改角色名称 / 描述，支持删除自定义角色（默认角色不可删除）

    -   权限分配：为不同角色分配**精细化操作权限**（如院级管理员仅可管理本院师生 / 比赛，奖品管理员仅可管理积分商城奖品 / 订单），支持**批量分配 / 取消权限**，权限按模块划分（用户管理 / 比赛管理 / 积分管理 / 奖品管理 / 系统管理），支持勾选 / 取消勾选单个权限

    -   角色关联：支持将任意用户关联至指定角色（可关联多个角色），支持批量关联，关联后用户立即拥有对应角色的权限，支持解除角色关联

3.  **全局比赛管理（总控）**

    -   比赛查看：展示平台**所有比赛**的完整信息（基础信息 / 时间地点 / 参与设置 / 作品设置 / 奖励设置 / 发起人 / 报名人数 / 签到率 / 作品上传率 / 评审进度），支持按发起人 / 比赛状态 / 难度 / 重要性 / 校区 / 发布时间筛选、搜索

    -   比赛操作：支持**强制下架违规比赛**（需填写原因，通知发起人 + 已报名学生）、**修改比赛关键信息**（紧急情况，如比赛时间 / 地点变更，需填写原因）、**置顶优质比赛**（同步至小程序首页置顶展示，可设置置顶时间）、**推荐比赛**（同步至小程序首页 “为你推荐” 专区）

    -   比赛数据：查看所有比赛的详细数据（报名 / 签到 / 作品 / 评审 / 获奖 / 积分发放），支持导出比赛所有数据，支持查看比赛日志（发布 / 编辑 / 下架 / 评分 / 发奖记录）

4.  **奖品与订单管理（全流程）**

    -   奖品管理：支持**新增 / 编辑 / 删除**积分商城奖品，设置奖品名称 / 所需积分 / 库存数量 / 兑换说明 / 奖品图片 / 领取方式 / 兑换限制 / 积分 + 现金比例，支持**批量导入 / 导出奖品**（模板导入），支持**上下架奖品**（下架后奖品从商城隐藏，已兑换订单正常处理）

    -   订单管理：展示所有积分兑换订单的完整信息（订单号 / 用户信息 / 奖品信息 / 兑换时间 / 所需积分 / 订单状态 / 领取方式 / 物流信息），支持按订单号 / 用户名 / 奖品名称 / 订单状态 / 兑换时间筛选、搜索；支持**更新订单状态**（待领取→已领取 / 已作废）、**录入物流信息**（线上邮寄订单）、**处理售后申请**（审核通过 / 驳回，需填写原因），支持导出订单列表

    -   库存管理：实时展示奖品库存数量，库存不足时自动提醒（后台红点 + 微信通知管理员），支持批量补货 / 减库存，支持设置库存预警值（低于预警值自动提醒）

5.  **系统管理（全功能）**

    -   日志管理：展示**所有系统操作日志**（用户注册 / 登录 / 账号操作 / 比赛发布 / 编辑 / 下架 / 积分发放 / 扣除 / 调整 / 奖品兑换 / 订单操作 / 权限分配），包含操作人 / 操作时间 / 操作内容 / IP 地址 / 操作结果，支持按操作人 / 操作类型 / 操作时间 / IP 地址筛选、搜索，支持导出日志、删除日志（保留近 3 年日志）

    -   轮播图管理：配置小程序首页轮播图，支持**上传 / 删除 / 排序**轮播图，设置轮播图跳转链接（如比赛详情 / 积分商城 / 活动页面），设置轮播图展示时间，支持**上下架轮播图**

    -   系统设置：设置小程序**名称 / LOGO / 副标题 / 客服联系方式 / 关于我们 / 隐私政策**；开启 / 关闭小程序部分功能（如积分商城 / 每日签到 / 比赛分享 / 作品上传）；设置小程序运行参数（如图片 / 文件大小限制、短信验证码有效期、密码规则）；清理系统缓存、备份系统数据（手动 / 自动备份，保留近 3 个月备份数据）

    -   多校区管理：支持**新增 / 编辑 / 删除**校区，设置校区名称 / 校区地址 / 校区管理员，支持将师生 / 比赛关联至指定校区，实现多校区独立管理与数据统计

6.  **数据统计与分析（全维度）**

    -   数据报表：生成**多维度数据报表**（用户报表 / 比赛报表 / 积分报表 / 奖品报表 / 订单报表），支持按日 / 周 / 月 / 季 / 年生成，支持自定义报表字段

    -   趋势分析：展示所有核心数据的**趋势图**（新增用户 / 比赛数 / 报名人数 / 积分发放 / 奖品兑换），支持对比不同时间段数据，支持查看数据同比 / 环比

    -   数据导出：所有数据报表 / 趋势图支持导出为**Excel / 图片 / PDF**格式，支持批量导出

    -   自定义统计：支持管理员**自定义统计条件**，生成个性化数据统计报表，支持保存统计模板（后续可直接使用）

十、特色功能模块 - 每日签到与比赛分享

核心需求

打造轻量化日常互动功能，提升小程序日活，增加学生参赛与分享意愿，丰富积分获取渠道。

功能详情

1.  **每日签到（学生端）**

    -   签到入口：小程序首页 / 个人中心醒目展示 “每日签到” 按钮，未签到时显示红色角标

    -   签到规则：每日可签到 1 次，点击按钮自动签到，发放固定积分（管理后台可配置）；**连续签到可叠加积分**（如连续签到 7 天，第 7 天发放双倍积分，规则可配置）；签到后显示 “今日已签到” 与连续签到天数，同步更新积分

    -   签到记录：支持查看历史签到记录（签到时间 / 获得积分 / 连续签到天数），支持按时间筛选

2.  **比赛分享（学生端）**

    -   分享入口：比赛详情页 / 个人中心 “我的比赛” 展示 “分享比赛” 按钮，支持分享至**微信好友 / 微信朋友圈**

    -   分享规则：每日限分享 1 次（任意比赛），分享成功后自动发放固定积分（管理后台可配置），积分实时到账，发送通知；朋友圈分享需截图上传验证（可选，管理后台可配置）

    -   分享奖励：支持设置**分享邀新奖励**（如分享后好友报名比赛，分享者可获得额外积分，规则可配置）

产品交互与体验要求

1.  **页面跳转**：所有页面跳转响应时间≤1 秒，支持**返回上一页、首页一键返回、页面缓存**，避免多层级跳转导致的操作繁琐；所有列表页支持下拉刷新、上拉加载更多

2.  **操作反馈**：所有点击、提交、报名、签到、上传、兑换等操作，均有**明确的文字 + 图标反馈**（如 “报名成功”“签到成功”“积分到账”），失败操作需提示**具体原因**（如 “名额已满”“签到已截止”“积分不足”），加载状态展示加载动画，避免页面空白

3.  **兼容性**：支持微信小程序最新版本，兼容主流手机系统（iOS 12.0 及以上、Android 8.0 及以上），适配不同屏幕尺寸（手机 / 平板），支持横竖屏切换；所有图片 / 视频自适应屏幕，无变形、模糊

4.  **易用性**：页面布局简洁明了，**核心功能入口突出**（如报名 / 签到 / 上传 / 兑换按钮为醒目颜色），操作步骤≤3 步；老师发布比赛的表单支持**分步填写、自动保存、必填项标注**，降低填写难度；所有表单支持一键清空、重置

5.  **可视化体验**：所有数据（积分 / 比赛 / 奖品 / 订单）均采用**图表 + 数字**可视化展示，支持切换图表类型（饼图 / 柱状图 / 折线图 / 热力图）；所有列表支持**排序 / 筛选 / 搜索**，支持自定义列表展示字段

6.  **多渠道提醒**：所有重要操作（报名 / 签到 / 积分发放 / 奖品兑换 / 比赛结果）均采用**小程序红点 + 微信服务通知**双重提醒，确保用户及时知晓；通知内容简洁明了，带相关链接，可一键跳转

7.  **文件 / 图片体验**：支持**在线预览**常见格式的文件 / 图片 / 视频 / 音频，支持**下载 / 保存**，支持**批量上传 / 下载**文件；图片上传支持裁剪 / 压缩，视频 / 音频上传支持进度展示

8.  **错误处理**：网络异常时提示 “网络错误，请检查网络后重试”，支持**重新加载**；操作超时 / 服务器错误时提示 “操作失败，请稍后再试”，支持**一键重试**；表单填写错误时，**实时提示错误位置与原因**，不通过的字段标红

非功能需求

一、性能需求

1.  小程序启动时间≤3 秒，首次加载资源≤5M，二次加载时间≤1 秒

2.  支持高并发访问，同时在线人数≥2000 人时，系统无卡顿、无崩溃、无数据丢失

3.  数据查询（如比赛列表 / 积分记录 / 订单列表）响应时间≤1 秒，复杂数据统计（如大屏数据）响应时间≤3 秒

4.  文件 / 图片 / 视频上传下载速度：单文件≤100M 时，上传 / 下载时间≤30 秒，支持断点续传

5.  数据导出（Excel / 图片 / PDF）响应时间≤10 秒，大文件（≥10 万条数据）导出时间≤30 秒，支持后台导出（导出完成后通知）

6.  小程序运行内存占用≤200M，无内存泄漏，长时间运行无卡顿

二、安全需求

1.  **数据加密**：用户学号、手机号、密码、收货地址等敏感信息采用**MD5 + 盐值加密**存储，传输过程采用 HTTPS 加密，防止数据泄露；所有积分 / 订单 / 比赛数据采用加密存储，防止篡改

2.  **接口防护**：所有接口请求做**鉴权处理**（用户 token 验证），防止非法请求、跨域请求、伪造请求；对恶意攻击（如 SQL 注入、XSS 攻击、CSRF 攻击、暴力破解）做防护，触发攻击后自动封禁 IP（可配置封禁时间）

3.  **操作权限**：严格的角色权限控制，**越权操作直接拒绝**，并记录入系统日志；所有关键操作（积分调整 / 账号冻结 / 比赛下架 / 权限分配）需二次确认，部分操作需超级管理员审批

4.  **防刷处理**：对每日签到、比赛分享、短信验证码等功能做**防刷处理**，限制同一账号 / IP 短时间内多次操作（可配置限制次数 / 时间），触发后暂时封禁功能（可配置封禁时间）

5.  **日志追溯**：所有系统操作、用户操作均记录**详细日志**，包含操作人 / 时间 / 内容 / IP / 结果，日志永久保存（可配置保留时间），支持一键追溯，便于问题排查与责任认定

6.  **数据备份**：系统数据每日**自动备份**（凌晨 2 点），支持手动备份，备份数据存储在独立服务器，保留近 3 个月备份数据；数据恢复支持按时间点恢复，恢复过程不影响小程序正常运行

7.  **内容审核**：比赛名称 / 说明 / 封面图、学生上传的作品、私信聊天内容等**支持敏感内容审核**（关键词 + 图片识别），敏感内容自动屏蔽 / 删除，触发后通知管理员与用户，计入违规记录

三、可维护性需求

1.  **代码规范**：前后端代码遵循**模块化、组件化开发**，增加详细注释（关键代码 / 复杂逻辑），代码结构清晰，便于后续功能迭代与 BUG 修复；采用统一的代码规范与命名规则

2.  **数据库设计**：数据库设计合理，**建立合理的索引**（提升查询速度），减少数据冗余，采用分表分库设计（针对大数据量，如用户表 / 比赛表 / 订单表），支持数据库读写分离

3.  **接口规范**：前后端接口采用**RESTful API**规范，接口文档完整、清晰，包含接口地址 / 请求方式 / 参数 / 返回值 / 错误码，支持接口版本管理，便于接口升级与兼容

4.  **系统监控**：搭建系统监控平台，实时监控小程序**运行状态、服务器状态、数据库状态、接口调用状态**，出现异常（如服务器宕机、数据库连接失败、接口调用失败）时，**立即向管理员发送报警通知**（微信 + 短信）

5.  **问题排查**：系统提供**完善的错误日志与调试工具**，便于开发人员排查问题；管理员后台提供**问题排查入口**，可查看实时运行日志、错误记录

6.  **版本管理**：小程序与管理后台采用**版本化管理**，支持版本升级、回滚，升级过程不影响用户使用，支持灰度发布（按校区 / 用户比例发布）

四、扩展性需求

1.  **功能接口预留**：预留丰富的功能接口，便于后续新增功能，如**比赛线上投票 / 评选、直播比赛、比赛题库、线上考试、校企合作比赛、赛事报名收费、电子证书认证**等

2.  **系统对接**：支持与**校园教务系统、一卡通系统、校园官网、微信公众号**对接，实现学号自动校验、线下奖品领取核销、校园信息同步、公众号消息推送、扫码登录等

3.  **多端适配**：支持将小程序功能适配至**微信公众号、H5、APP**，实现多端数据同步、账号互通，打造全端一体化平台

4.  **第三方集成**：支持集成 \*\* 第三方支付（微信支付 / 支付宝）、第三方物流（顺丰 / 圆通 / 中通）、第三方存储（阿里云 / 腾讯云）、第三方审核（百度 AI / 阿里云 AI）\*\* 等服务，提升平台功能与体验

5.  **国际化适配**：支持**多语言切换**（中文 / 英文），适配不同地区的时间 / 时区 / 货币，便于后续拓展至海外校园

6.  **个性化定制**：支持为**不同校区 / 学校**提供个性化定制服务，如自定义小程序 LOGO / 名称 / 配色、自定义功能模块、自定义积分规则 / 奖品类型，实现一平台多校区管理
