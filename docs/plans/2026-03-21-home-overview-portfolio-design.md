# 首页进度卡片与我的成果页设计

**问题背景**

首页“我的进度”区域当前只是数字展示，用户点击后没有任何反馈，无法从“我的比赛 / 已交作品 / 获奖次数 / 总积分”直接进入对应功能。现有小程序里虽然已经有比赛列表、积分页、结果页、作品上传页，但缺少一个面向个人的“我的作品 / 我的成果”聚合视图。

**目标**

1. 首页中间四张进度卡片都支持点击跳转。
2. `我的比赛` 和 `总积分` 继续复用现有页面。
3. 新增一个“我的成果”页，承接 `已交作品` 与 `获奖次数` 两张卡片。
4. 页面尽量复用现有数据接口，只为“按用户查询全部作品记录”补一个最小后端接口。

**方案对比**

1. 只做卡片跳转，全部复用现有页面。  
   优点是开发快。  
   缺点是 `已交作品` 没有精准承接页，用户体验别扭。

2. 新增一个轻量“我的成果”页，推荐。  
   优点是结构清晰，能同时承接作品和获奖信息；只需补一个用户作品列表接口。  
   缺点是需要新增页面与少量后端能力。

3. 直接做完整“我的作品管理中心”。  
   优点是信息最全。  
   缺点是会引入筛选、分页、版本管理等超出当前需求的复杂度，不适合这一轮。

**最终设计**

- 首页进度卡片改为可点击卡片：
  - `我的比赛` -> `/pages/competition/list/index`
  - `已交作品` -> `/pages/competition/portfolio/index?tab=works`
  - `获奖次数` -> `/pages/competition/portfolio/index?tab=awards`
  - `总积分` -> `/pages/points/index`
- 新增 `pages/competition/portfolio` 页面：
  - 顶部展示“我的成果”标题与说明
  - 作品区：展示当前用户提交过的作品记录，包含比赛名、版本号、提交时间、文件名，并提供“查看该比赛作品页”入口
  - 获奖区：展示当前用户获奖结果，包含比赛名、奖项、名次、积分、发布时间，并提供“查看比赛结果”入口
  - 若通过 `tab=works` / `tab=awards` 进入，则高亮对应分区标题并优先呈现该内容
- 后端补充只读接口：
  - `GET /api/app/submissions/user/{userId}`
  - 使用现有 `submissionService.listByUser(userId)` 返回当前用户作品记录
  - 继续走 `AuthContext.requireUser(userId)` 权限校验

**数据来源**

- 作品数据：新增小程序服务 `fetchUserSubmissions(userId)`，请求新接口。
- 获奖数据：继续复用 `fetchStudentOverview(userId)` 中已有 `results`。
- 比赛标题：复用 `fetchCompetitions()` 提供的比赛列表做 `competitionId -> title` 映射。

**测试策略**

- 后端：补服务层测试，确认 `listByUser` 结果可供个人成果页使用；控制器只做最小代码变更，避免扩大集成测试范围。
- 小程序：
  - 补 helper 测试，锁住首页卡片路由与成果页数据整理函数
  - 补源码测试，确认首页卡片具备点击行为、新页面已注册且使用正确服务
- 回归：
  - 跑小程序全量 Node 测试
  - 跑后台全量测试，确保这次改动没有意外影响共享工具
