# 后台创建用户与双端界面收敛设计

## 背景

当前项目已经具备基础的后台登录、管理壳子和原生微信小程序登录能力，但账号流程与界面体验仍存在明显缺口：

1. 用户端 `wechat-native` 没有注册页，但后端存在 `/api/app/auth/register`，账号来源对最终交付并不清晰。
2. 后台用户治理页只支持查看、冻结、解冻、重置密码和分配角色，缺少“创建用户”与“违规标记/解除”完整闭环。
3. `wechat-native` 的登录页、首页、个人中心信息块偏多，页面视觉层级散，成品感不足。
4. 后台整体已有左右结构，但用户治理与壳层视觉仍偏毛坯，不够统一。

## 目标

本轮改造聚焦三个目标：

- 明确账号流程为“后台创建用户，用户端只登录不注册”
- 补齐后台创建用户与违规治理闭环
- 对后台和 `wechat-native` 做一次减法式美化，提升成品感

## 采用方案

采用“账号流程收口 + 关键页面收敛”的稳妥方案：

- 后台增加管理员创建用户接口与页面入口
- 用户端不新增注册入口，只保留老师/学生登录
- 界面优化优先覆盖高频入口，不重做全部页面

这样可以在不推翻现有业务链路的前提下，把体验和交付完整度拉起来。

## 账号流程设计

### 1. 后台创建用户

管理员在后台用户治理页填写：

- 学号
- 姓名
- 手机号
- 角色
- 初始密码

提交后后端写入 `sys_user`，状态默认 `ENABLED`，密码使用现有 BCrypt 方案加密。

### 2. 用户登录

用户在 `wechat-native` 登录页输入：

- 身份（学生 / 老师）
- 学号
- 密码

登录成功后沿用现有 token 会话逻辑。登录页文案将明确说明：

- 账号由管理员开通
- 首次登录后可在个人中心修改密码

### 3. 注册说明

后端虽然保留 `/api/app/auth/register`，但本期不在用户端暴露入口，也不作为正式交付流程。

对外说明统一为：

> 本系统采用后台开通账号模式，用户端只负责登录和后续资料维护。

## 后端设计

### 1. 新增管理员创建用户接口

新增接口：

- `POST /api/admin/users`

请求参数：

- `studentNo`
- `realName`
- `phone`
- `roleCode`
- `password`

约束：

- 学号不能为空且唯一
- 姓名不能为空
- 密码长度不少于 8 位
- 角色必须是系统已有角色

返回值：

- 新创建的 `UserSummary`

### 2. 操作日志

创建用户成功后写入后台操作日志，动作建议为：

- `USER_CREATE`

目标记录为新建账号学号，详情记录角色信息。

### 3. 安全处理

- 继续使用参数化 ORM 查询，不拼接 SQL
- 不在接口响应中返回密码或密码哈希
- 前端表单只做基础校验，最终约束以后端为准

## 后台页面设计

### 1. 用户治理页

页面拆成三块：

- 左侧：用户列表
- 右侧上方：选中用户治理面板
- 右侧下方：新建用户面板

在现有能力基础上补充：

- 创建用户
- 违规标记
- 解除违规标记

成功操作后刷新列表并保持当前选中状态。

### 2. 后台壳层美化

保留现有左右结构，增强：

- 左侧导航层次与选中态
- 顶部标题区信息分组
- 主内容区卡片背景、边框、阴影与留白

风格方向保持为暖米色 + 冷灰蓝的治理后台，不做大幅重构。

## wechat-native 页面设计

### 1. 登录页

保留当前身份选择和演示账号快捷带入，但优化为更聚焦的登录卡片：

- 弱化多余说明
- 强化“账号由管理员创建”
- 优化按钮和输入框层次

### 2. 首页

首页收敛为三层：

- 欢迎与账号概览
- 今日任务与快捷操作
- 核心功能入口

不再堆叠重复说明文案，只保留当前角色最常用的信息。

### 3. 个人中心

个人中心改为“主功能优先，低频功能后置”的结构：

- 账号概览
- 基础资料
- 密码修改
- 提醒与隐私
- 意见反馈
- 账号注销

同时压缩说明性文本，减少一屏信息密度。

## 测试策略

### 后端

- 新增管理员创建用户 API 测试
- 新增管理员创建用户服务/持久化测试

### 后台前端

- 新增创建用户 API client 测试
- 增加用户治理相关表单校验与行为测试

### 用户端

`wechat-native` 当前以页面逻辑为主，本轮优先做运行层验证，不额外引入新的测试框架。

## 验收标准

- 后台可创建学生/老师/管理员账号
- 新建账号后可在用户列表中立即看到，并能继续执行冻结、重置密码、改角色和违规治理
- `wechat-native` 登录页明确体现“仅登录，不注册”
- `wechat-native` 首页和个人中心信息更聚焦、视觉更统一
- 后台用户治理页和整体壳层视觉更完整
- 后端与后台测试通过
