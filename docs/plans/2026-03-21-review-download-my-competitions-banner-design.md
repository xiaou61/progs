# 评审下载、我的比赛筛选与首页轮播图设计

## 背景

当前系统有 3 个直接影响使用体验的问题：

1. 评审工作台只能选作品编号，拿不到作品文件地址，老师无法直接下载评审文件。
2. 小程序首页“我的比赛”卡片虽然显示的是我报名的数量，但点击后进入的是公开比赛列表，看到的是全部比赛。
3. 后台已有轮播图管理页，但小程序首页没有读取或展示任何轮播图，所以后台配置对移动端无感。

## 目标

一次性打通以下体验：

1. 评审老师在工作台中点击作品即可直接下载对应提交文件。
2. 小程序首页“我的比赛”点击后只显示当前用户已报名的比赛。
3. 小程序首页展示后台启用中的轮播图，点击后按后台配置跳转。

## 方案

### 1. 评审文件下载

- 扩展评审任务接口返回结构，在 `ReviewTaskSummary` 中补充 `fileUrl`、`fileName`、`versionNo`、`submittedAt`。
- `ReviewService.listTasks()` 直接从 `SubmissionSummary` 派生这些字段，不需要额外查表。
- 管理端 `ReviewWorkbenchPage.vue` 在作品下拉文案、待评作品卡片中显示文件名与版本信息。
- 点击作品卡片或“作品编号”对应条目时，直接触发下载：
  - 若 `fileUrl` 是本地相对路径，如 `/uploads/submissions/...`，则拼接当前站点源地址下载。
  - 保留现有“带入评审表单”行为，不与下载入口冲突。
- 交互上按照用户要求采用“点击作品就下载”，不单独增加“查看”按钮。

### 2. 小程序“我的比赛”只看已报名

- 小程序首页“我的比赛”跳转路径改为 `/pages/competition/list/index?scope=my`。
- 小程序比赛列表页读取 `scope`：
  - 默认 `scope=all`，维持现有公开比赛列表行为。
  - `scope=my` 时，先获取当前用户报名记录，再仅展示报名状态为 `REGISTERED` 的比赛。
- 为避免新建一页，继续复用现有比赛列表页，只增加筛选态和空状态文案。
- 同时保留首页统计卡数量来源不变，确保“数量”和“列表结果”一致。

### 3. 首页轮播图

- 后端新增小程序可读接口，例如 `GET /api/app/banners`，只返回 `ENABLED` 的轮播图。
- 小程序首页新增轮播图区域：
  - 读取后台启用轮播图列表。
  - 使用 `swiper` 展示标题和跳转提示。
  - 点击轮播图按 `jumpPath` 打开；如果配置缺失则不跳。
- 若没有启用轮播图，则首页不显示轮播区，避免占空。

## 数据流

### 评审下载

`cmp_submission` -> `SubmissionService.listByCompetition()` -> `ReviewService.listTasks()` -> `ReviewTaskSummary(fileUrl...)` -> 管理端评审工作台 -> 浏览器下载

### 我的比赛

首页卡片 -> 比赛列表页 `scope=my` -> 报名接口拿到当前用户报名记录 -> 过滤公开比赛列表 -> 只显示已报名项

### 轮播图

后台轮播图管理 -> `sys_banner` -> 小程序轮播图接口 -> 首页 `swiper` -> 点击按 `jumpPath` 跳转

## 错误处理

- 评审文件缺失时，下载动作给出“当前作品未上传文件地址”的提示。
- “我的比赛”无报名记录时，显示“你还没有报名任何比赛”。
- 轮播图接口失败时，不阻塞首页其他模块，只静默隐藏轮播区并保留日志/错误提示位。

## 测试策略

- 后端：
  - 评审任务接口返回作品文件字段。
  - 小程序轮播图接口仅返回启用项。
- 管理端：
  - 评审 API 客户端包含 `fileUrl/fileName`。
  - 评审工作台源码存在作品下载入口与文件信息展示。
- 小程序：
  - 首页“我的比赛”跳转带 `scope=my`。
  - 比赛列表页支持 `scope=my` 过滤逻辑。
  - 首页接入轮播图加载、展示和点击跳转。

## 取舍

- 不单独新建“我的比赛”页面，优先复用现有比赛列表页，降低修改范围。
- 轮播图只做标题型轮播，不在本轮新增图片上传能力，避免扩散到素材管理。
- 下载采用浏览器/当前宿主原生下载能力，不新增后端文件代理。
