# 微信老师发布时间选择器与作品上传引导设计

## 背景

当前微信小程序老师端比赛发布页要求老师手动输入 `YYYY-MM-DDTHH:mm:ss` 格式时间，输入成本高且容易出错。学生端作品上传页虽然已经接入真实上传接口，但缺少明确的步骤提示，老师和学生都不容易第一眼理解“先选文件、再确认、再提交”的流程。

## 目标

1. 老师端比赛发布页中所有和时间有关的字段都改成选择器交互，不再要求手动输入时间字符串。
2. 提交给后端的时间格式保持不变，继续传 `YYYY-MM-DDTHH:mm:ss`，避免联动改后端接口。
3. 作品上传页改成更明确的步骤式引导，让“选择文件”和“提交作品”两个动作更容易理解。

## 方案选择

### 方案 A：原生日期选择 + 时间选择

- 每个时间字段拆成“日期选择器 + 时间选择器”。
- 页面显示友好格式，内部自动拼成接口需要的 ISO 风格字符串。
- 保持现有表单结构和校验逻辑，只增强交互体验。

这是本次采用的方案。优点是改动集中、风险低、兼容现有接口。

### 方案 B：自定义一体化日期时间面板

- 自己组合弹层或复杂联动控件，一次选完整日期时间。

不采用。原因是小程序端实现与维护成本明显更高，本轮需求没有必要引入额外复杂度。

## 详细设计

### 一、老师端比赛发布页

- 保留原有四个字段：`signupStartAt`、`signupEndAt`、`startAt`、`endAt`。
- 页面层不再使用 `<input>` 手输时间，改成统一的时间选择区块：
  - 第一行显示字段名称
  - 第二行提供日期选择器
  - 第三行提供时间选择器
  - 底部展示已选中的完整时间
- 当字段为空时，给出可理解的默认显示文案，避免空白状态。
- JS 层新增时间工具方法：
  - 把现有 `YYYY-MM-DDTHH:mm:ss` 拆成日期和时间
  - 把选择器产出的日期和时间重新拼成接口需要的格式
  - 格式化页面展示文案

### 二、学生端作品上传页

- 页面顶部增加三步说明：
  - 第 1 步：选择本地作品文件
  - 第 2 步：确认文件信息与覆盖方式
  - 第 3 步：提交作品并查看版本记录
- 已选文件区域增加更清晰的状态文案，明确区分“未选择”“已选择待提交”。
- 版本记录区域增加说明，强调最新版本会作为评审依据。
- 主按钮文案调整为更明确的提交动作，减少“只上传文件不提交记录”的误解。

## 数据与兼容性

- 后端接口、字段名、校验规则保持不变。
- 小程序页面仅调整展示结构和前端格式转换。
- 现有列表加载、编辑回显、发布/草稿保存、作品上传接口都继续沿用。

## 测试策略

1. 为时间工具新增单元测试，验证拆分、拼装、显示格式。
2. 为老师端页面新增源码测试，验证手输提示已移除、选择器文案已存在。
3. 为作品上传页新增源码测试，验证三步提示和更直白的上传状态文案存在。
