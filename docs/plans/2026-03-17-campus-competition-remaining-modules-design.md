# 师生比赛管理平台剩余模块设计

## 1. 目标

在现有 MVP 主链路基础上，继续完成 PRD 中尚未落地、且不依赖第三方能力的剩余模块，形成一个可本地完整运行的校园比赛管理平台增强版。

## 2. 范围裁剪原则

本轮严格去掉所有必须依赖第三方服务的能力，避免引入短信、微信通知、支付、物流、AI 审核、外部系统对接等外部依赖。

### 2.1 明确不做

- 短信验证码注册、短信找回密码
- 微信服务通知
- 微信支付 / 支付宝支付
- 第三方物流状态同步
- 百度 AI / 阿里云 AI 审核
- 教务系统、一卡通、校园官网、公众号等外部系统对接
- 多校区高级隔离模型
- 积分商城与订单

### 2.2 保留并落地

- 账号与个人中心
- 比赛管理增强
- 学生端参与增强
- 消息中心
- 后台权限与治理
- 数据大屏
- 本地规则内容审核

## 3. 剩余模块清单

## 3.1 账号与个人中心

- 个人资料查询与编辑
- 修改密码
- 账号注销
- 消息提醒设置
- 隐私设置
- 意见反馈
- 我的比赛、我的作品、获奖记录、个人数据汇总

说明：

- 忘记密码改为后台重置密码和站内修改密码，不再做短信找回。
- 注销采用“不可恢复停用账号”方案，保留必要比赛历史，清空可编辑资料和登录能力。

## 3.2 比赛管理增强

- 老师草稿箱
- 比赛编辑与下架
- 更完整的比赛字段
- 自定义报名项
- 报名驳回、手动添加参赛人、到场/缺席标记
- 签到统计与导出
- 批量评审、评审意见、多评委基础版
- 排名发布增强
- 固定模板电子奖状生成、预览、查看、保存

说明：

- 不做微信通知和群通知推送，但所有变更都会写入系统消息中心。
- 奖状采用固定 HTML/Canvas 模板生成图片文件，不依赖外部服务。

## 3.3 学生端参与增强

- 比赛筛选、排序、收藏、推荐
- 报名凭证
- 每日签到任务
- 比赛分享入口
- 奖状中心

说明：

- 比赛分享只保留本地交互与积分发放入口，不做微信真实分享回调校验。
- 每日签到和比赛分享的积分发放均由本地规则完成。

## 3.4 消息中心

- 系统消息
- 一对一私信
- 比赛群聊
- 会话列表、未读数、置顶、免打扰、清空
- 比赛详情页一键咨询老师

说明：

- 消息只做站内消息，不做微信服务通知。
- 群聊按“每场比赛一个群”建模，报名成功后自动加入。

## 3.5 后台权限与治理

- 用户冻结 / 解冻
- 重置密码
- 违规标记
- 角色管理
- 权限分配
- 用户角色关联
- 比赛总控、推荐/置顶、比赛日志
- 更完整的系统设置、轮播图、功能开关
- 报表导出

说明：

- 多校区管理保留当前基础兼容结构，但本轮不继续增强。
- 权限先做到菜单级、接口级、按钮级三层控制。

## 3.6 数据大屏

- 管理后台汇总大屏
- 老师个人比赛大屏
- 时间范围筛选
- 手动刷新
- Excel / 图片 / PDF 导出

说明：

- 统计只围绕本地已有比赛、积分、消息、用户数据生成。
- 不做多校区和商城维度统计。

## 3.7 本地规则内容审核

- 比赛标题 / 描述敏感词检测
- 作品文件名 / 地址规则审核
- 私信与群聊敏感词拦截
- 违规记录入库与后台查看

说明：

- 仅做关键词规则引擎和本地审计记录，不做图片识别和第三方审核。

## 4. 推荐实施顺序

### Phase A：账号与个人中心 + 比赛管理增强

- 先补齐用户资料、密码、注销、设置
- 再补老师草稿箱、编辑/下架、自定义报名项、报名管理
- 最后补奖状和比赛增强能力

### Phase B：消息中心

- 先做系统消息
- 再做私信会话
- 再做比赛群聊和消息设置

### Phase C：后台权限与治理

- 先做角色、权限、用户治理
- 再做比赛总控、日志、报表

### Phase D：数据大屏

- 先做后台汇总
- 再做老师个人大屏
- 再做导出

### Phase E：本地规则内容审核

- 先做敏感词规则
- 再接比赛、作品、聊天拦截
- 最后补违规记录和后台查看

## 5. 实现原则

- 所有新增能力都必须走 TDD
- 优先复用现有主链路的真实数据，而不是新造平行模型
- 所有剩余模块都必须是“无第三方依赖也能运行”的版本
- 功能状态一律明确为：已完成 / 本地版 / 明确不做
- 先做可用闭环，再做体验增强

## 6. 当前结论

在“去掉第三方能力、去掉积分商城与多校区”的前提下，剩余 PRD 的主要实现范围已经明确，可以进入新一轮分批实现。
