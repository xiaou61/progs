# MVP 验收手册

## 目标

验证 MVP 版本已具备从比赛发布到结果发布、积分入账的最小闭环能力。

## 验收范围

1. 管理后台可进入用户、角色、校区、比赛发布、评审工作台、轮播图、系统配置页面。
2. 小程序可进入登录、首页、比赛列表、比赛详情、报名、签到、作品上传、结果、积分页面。
3. 后端可提供健康检查、比赛、报名、签到、作品、评分、积分、运营配置接口。

## 验收步骤

1. 执行 `mvn -f server/pom.xml test`，确认服务端测试全部通过。
2. 执行 `pnpm --dir admin-web test -- --run` 与 `pnpm --dir admin-web build`，确认后台测试和构建通过。
3. 执行 `pnpm --dir miniapp test -- --run` 与 `pnpm --dir miniapp build:h5`，确认小程序测试和 H5 构建通过。
4. 启动后端服务，访问 `/api/public/health`，确认返回 `code=0`。
5. 使用后台接口发布比赛，使用小程序接口完成报名、签到、作品提交。
6. 使用后台结果发布接口为学生发放积分，并通过小程序结果页接口检查积分到账。

## 已知限制

- 当前为 MVP 骨架，业务数据以内存态服务为主，服务重启后数据不会保留。
- 微信登录、真实文件上传、消息通知、商城兑换、多校区隔离仍在后续阶段。
